Nossa Agenda API

API REST completa para o sistema de agenda compartilhada para casais. Autenticação por cookie HTTP-only (JWT), SQLite via Prisma ORM, e envio de emails com Resend.

BASE URL http://localhost:3000/api
Framework
Next.js 16
App Router · Route Handlers
Autenticação
JWT + Cookie
HTTP-only · SameSite=Lax · 7 dias
Banco de Dados
SQLite / Prisma
Prisma ORM 6 · Migrations
Email
Resend SDK
Convites de parceiro · HTML templates
Validação
Zod
Schema validation em todas as rotas
Senha
bcryptjs
Salt rounds: 12

🔐 Autenticação

O sistema usa cookies HTTP-only para manter a sessão. Não é necessário enviar tokens nos headers.

🔄 Fluxo de Autenticação
POST /auth/register
→
Cookie agenda_session definido
→
Redirect /agenda
POST /auth/login
→
Verifica credenciais
→
Cookie definido
→
Redirect /agenda
GET /auth/me
→
Lê cookie + verifica JWT
→
Retorna user ou null
ℹ️
O cookie agenda_session é definido automaticamente após login/registro. Para testar via curl, adicione -c cookies.txt -b cookies.txt.

📡 Endpoints

POST /api/auth/register público Criar conta + casal ▼
✅ Cria um novo usuário e um novo casal simultaneamente. O usuário é definido como partner1 do casal.
Body (application/json ou application/x-www-form-urlencoded)
CampoTipoValidaçãoObrigatório
email string Formato de email válido sim
password string Mínimo 6 caracteres sim
name string Mínimo 2 caracteres sim
coupleName string Mínimo 2 caracteres. Ex: "João & Maria" sim
Exemplo de requisição
curl
JavaScript
curl -X POST http://localhost:3000/api/auth/register \
  -H "Content-Type: application/json" \
  -c cookies.txt \
  -d '{
    "email": "joao@casal.com",
    "password": "minhasenha123",
    "name": "João",
    "coupleName": "João & Maria"
  }'
Respostas
302 Redireciona para /agenda com cookie definido
400 Dados inválidos (validação Zod falhou)
409 Email já cadastrado
POST /api/auth/login público Autenticar usuário ▼
Body
CampoTipoValidaçãoObrigatório
email string Formato de email válido sim
password string Não vazio sim
Exemplo
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -c cookies.txt -b cookies.txt \
  -d '{"email":"joao@casal.com","password":"minhasenha123"}'
Respostas
302 Redirect para /agenda — cookie agenda_session definido (7 dias)
401 Email ou senha incorretos
POST /api/auth/logout autenticado Encerrar sessão ▼
⚠️ Remove o cookie agenda_session e retorna {"ok": true}. 
curl -X POST http://localhost:3000/api/auth/logout \
  -b cookies.txt
Resposta 200
{ "ok": true }
GET /api/auth/me autenticado Dados da sessão atual ▼
ℹ️ Retorna dados completos do usuário autenticado incluindo informações do casal e parceiros. 
curl http://localhost:3000/api/auth/me \
  -b cookies.txt
Resposta 200 — Autenticado
{ "user": { "id": "clx...", "email": "joao@casal.com", "name": "João", "coupleId": "clx...", "coupleName": "João & Maria", "couple": { "name": "João & Maria", "partner1": "João", "partner2": "Maria", "startDate": "2023-02-14" } } }
Resposta 200 — Não autenticado
{ "user": null }
POST /api/auth/invite-partner autenticado Convidar parceiro(a) ▼
ℹ️ Cria um token de convite (expira em 7 dias) e opcionalmente envia email via Resend. Se RESEND_API_KEY não estiver configurado, retorna o link no corpo da resposta.
Body
CampoTipoValidaçãoObrigatório
partnerEmail string Email válido + não cadastrado sim 
curl -X POST http://localhost:3000/api/auth/invite-partner \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{"partnerEmail":"maria@casal.com"}'
Resposta 200
{ "message": "Convite enviado por email!", "inviteLink": "http://localhost:3000/aceitar-convite?token=abc...", "emailSent": true }
Erros possíveis
400 Parceira já vinculada ao casal
400 Email já possui conta — redirecionar para login
401 Não autenticado
GET /api/auth/accept-invite?token=<token> público Validar token de convite ▼
Query Params
ParamTipoDescrição
token string Token de 64 chars gerado em /invite-partner
Resposta 200 — Token válido
{ "email": "maria@casal.com", "coupleName": "João & Maria", "valid": true }
POST /api/auth/accept-invite público Criar conta pelo convite ▼
Body
CampoTipoValidaçãoObrigatório
token string Token válido e não usado sim
password string Mínimo 6 caracteres sim
name string Mínimo 2 caracteres sim
Resposta 200
{ "user": { "id": "clx...", "email": "maria@casal.com", "name": "Maria", "coupleId": "clx...", "coupleName": "João & Maria" } }

⚙️ Variáveis de Ambiente

Configure no arquivo .env na raiz do projeto.

🗄️ Banco de Dados
DATABASE_URL obrigatório
Caminho para o banco SQLite.
"file:./prisma/dev.db"
🔐 JWT
JWT_SECRET obrigatório
Segredo para assinar tokens JWT. Use uma string longa e aleatória em produção.
openssl rand -base64 32
📧 Email (Resend)
RESEND_API_KEY opcional
Chave da API Resend. Sem ela, convites retornam apenas o link.
Obtenha em: resend.com/api-keys
EMAIL_FROM opcional
Remetente padrão.
"Nossa Agenda <onboarding@resend.dev>"
🌐 App URL
NEXT_PUBLIC_APP_URL obrigatório
URL base para gerar links nos emails.
Dev: http://localhost:3000
Arquivo .env completo
# Banco de Dados
DATABASE_URL="file:./prisma/dev.db"

# JWT — gere com: openssl rand -base64 32
JWT_SECRET="seu-segredo-super-secreto-mude-em-producao"

# Resend — https://resend.com/api-keys
RESEND_API_KEY="re_xxxxxxxxxxxxxxxxxxxx"
EMAIL_FROM="Nossa Agenda <onboarding@resend.dev>"

# URL do App
NEXT_PUBLIC_APP_URL="http://localhost:3000"

🗄️ Schema do Banco de Dados

Tabela
User
Usuários registrados
Tabela
Couple
Casais e configurações
Tabela
PartnerInvite
Convites com token
// prisma/schema.prisma

model User {
  id           String   @id @default(cuid())
  email        String   @unique
  passwordHash String
  name         String
  coupleId     String
  couple       Couple   @relation(fields: [coupleId], references: [id], onDelete: Cascade)
  createdAt    DateTime @default(now())
  updatedAt    DateTime @updatedAt
}

model Couple {
  id         String          @id @default(cuid())
  name       String
  partner1Id String?
  partner2Id String?
  startDate  String?
  users      User[]
  invites    PartnerInvite[]
  createdAt  DateTime        @default(now())
  updatedAt  DateTime        @updatedAt
}

model PartnerInvite {
  id        String    @id @default(cuid())
  email     String
  token     String    @unique   // 32 bytes hex = 64 chars
  coupleId  String
  couple    Couple    @relation(...)
  expiresAt DateTime           // now + 7 dias
  usedAt    DateTime?
  createdAt DateTime  @default(now())
}

📧 Configuração de Email

O sistema usa Resend para envio de emails. Configure em resend.com.

ℹ️
Sem RESEND_API_KEY, o sistema funciona normalmente — o link de convite é retornado na resposta da API para ser compartilhado manualmente.
Passos para configurar Resend
# 1. Criar conta em resend.com
# 2. Verificar domínio em resend.com/domains (para produção)
# 3. Gerar API Key em resend.com/api-keys
# 4. Adicionar ao .env:
RESEND_API_KEY="re_xxxxxxxxxxxxxxxxxxxx"
EMAIL_FROM="Nossa Agenda <convite@seudominio.com>"

# Para testes locais sem domínio próprio:
EMAIL_FROM="Nossa Agenda <onboarding@resend.dev>"

{ "error": "Descrição legível do erro", "details": { /* opcional — detalhes de validação Zod */ } }
